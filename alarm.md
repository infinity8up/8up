# 인앱 알림 목록

현재 사용자 앱 인앱 알림은 `notifications` 테이블 기반으로 동작한다.

## 배지 규칙

- 중요 미읽음 알림이 1개 이상 있으면 종 아이콘에 빨간 `!` 배지가 표시된다.
- 중요 알림은 없고 미읽음 알림만 있으면 파란 점 배지가 표시된다.

## 적용된 알림 목록

### 1. 스튜디오 전체 알림


| kind     | 알림명          | 중요 여부                     | 대상               | 발생 조건             |
| -------- | ------------ | ------------------------- | ---------------- | ----------------- |
| `notice` | 공지 / 중요 공지   | 공지의 `is_important` 값에 따름  | 해당 스튜디오 활성 회원 전체 | 공지 등록 또는 공개 전환 시  |
| `event`  | 이벤트 / 중요 이벤트 | 이벤트의 `is_important` 값에 따름 | 해당 스튜디오 활성 회원 전체 | 이벤트 등록 또는 공개 전환 시 |


### 2. 수업 변경 / 취소 관련 알림


| kind                          | 알림명   | 중요 여부 | 대상                     | 발생 조건                      |
| ----------------------------- | ----- | ----- | ---------------------- | -------------------------- |
| `session_cancelled`           | 수업 취소 | 중요    | 해당 수업 예약/대기/취소요청 중인 회원 | 스튜디오가 수업 자체를 취소한 경우        |
| `session_instructor_changed`  | 강사 변경 | 중요    | 해당 수업 예약/대기/취소요청 중인 회원 | 관리자 웹에서 수업 강사를 변경한 경우      |
| `session_reservation_removed` | 예약 변경 | 중요    | 해당 회원 1명               | 관리자가 특정 회원을 수업에서 직접 제외한 경우 |


### 3. 취소 문의 관련 알림


| kind                      | 알림명   | 중요 여부 | 대상       | 발생 조건         |
| ------------------------- | ----- | ----- | -------- | ------------- |
| `cancel_request_approved` | 취소 승인 | 일반    | 해당 회원 1명 | 취소 문의가 승인된 경우 |
| `cancel_request_rejected` | 취소 거절 | 일반    | 해당 회원 1명 | 취소 문의가 거절된 경우 |


비고:

- `cancel_request_approved` 본문에는 복원된 잔여 횟수가 포함된다.
- 관리자 코멘트가 있으면 승인/거절 알림 본문에 함께 포함된다.

### 4. 예약 / 대기 관련 알림


| kind                    | 알림명   | 중요 여부 | 대상       | 발생 조건                           |
| ----------------------- | ----- | ----- | -------- | ------------------------------- |
| `reservation_created`   | 예약 완료 | 일반    | 해당 회원 1명 | 사용자가 직접 예약했거나 관리자가 대신 예약한 경우    |
| `waitlist_registered`   | 대기 예약 | 일반    | 해당 회원 1명 | 사용자가 대기 등록했거나 관리자가 대신 대기 등록한 경우 |
| `waitlist_promoted`     | 예약 확정 | 일반    | 해당 회원 1명 | 대기 예약이 실제 예약으로 승급된 경우           |
| `reservation_cancelled` | 예약 취소 | 일반    | 해당 회원 1명 | 사용자가 직접 예약을 취소한 경우              |


비고:

- `reservation_created`, `waitlist_promoted`, `reservation_cancelled` 본문에는 현재 수강권 잔여 횟수가 포함된다.
- `waitlist_registered`는 현재 수강권 잔여 횟수를 함께 안내한다. 대기 등록 자체는 차감이 일어나지 않지만, 사용자가 현재 상태를 확인할 수 있도록 넣었다.

### 5. 수강권 관련 알림


| kind                   | 알림명    | 중요 여부 | 대상       | 발생 조건                 |
| ---------------------- | ------ | ----- | -------- | --------------------- |
| `pass_issued`          | 수강권 발급 | 일반    | 해당 회원 1명 | 관리자 웹에서 수강권을 발급한 경우   |
| `pass_refunded`        | 수강권 환불 | 일반    | 해당 회원 1명 | 관리자 웹에서 수강권 환불 처리한 경우 |
| `pass_hold_registered` | 수강권 홀딩 | 일반    | 해당 회원 1명 | 홀딩 등록 또는 홀딩 기간 변경 시   |
| `pass_hold_cancelled`  | 홀딩 취소  | 일반    | 해당 회원 1명 | 아직 시작되지 않은 홀딩을 취소한 경우 |
| `pass_hold_ended`      | 홀딩 종료  | 일반    | 해당 회원 1명 | 진행 중인 홀딩을 조기 종료한 경우   |


비고:

- `pass_issued` 본문에는 발급된 수강권명, 사용 기간, 현재 잔여 횟수가 포함된다.
- `pass_refunded`는 환불 사유가 있으면 본문에 함께 포함된다.

### 6. 회원 상태 관련 알림


| kind                            | 알림명   | 중요 여부 | 대상       | 발생 조건                             |
| ------------------------------- | ----- | ----- | -------- | --------------------------------- |
| `studio_membership_approved`    | 가입 승인 | 일반    | 해당 회원 1명 | 스튜디오에서 회원 등록을 승인하거나 신규 멤버로 추가한 경우 |
| `studio_membership_reactivated` | 이용 재개 | 일반    | 해당 회원 1명 | 비활성 회원을 다시 활성화한 경우                |


### 7. 리마인드 알림


| kind                           | 알림명     | 중요 여부 | 대상       | 발생 조건          |
| ------------------------------ | ------- | ----- | -------- | -------------- |
| `session_reminder_day_before`  | 하루 전 알림 | 일반    | 해당 회원 1명 | 수업 시작 약 24시간 전 |
| `session_reminder_hour_before` | 곧 시작    | 일반    | 해당 회원 1명 | 수업 시작 약 1시간 전  |


비고:

- 리마인드 알림은 `dispatch_scheduled_user_notifications()` 함수가 생성한다.com.
- 자동 생성은 `pg_cron` 확장이 가능한 환경에서 `setup_scheduled_user_notification_jobs()`가 등록될 때 동작한다.
- `pg_cron`이 없는 환경에서는 함수는 존재하지만 자동 스케줄은 등록되지 않는다.

## 잔여 횟수 알림 흐름 예시

- 10회권 발급: `pass_issued`
- 1회 예약: `reservation_created` 본문에 `9회 남음`
- 1회 추가 예약: `reservation_created` 본문에 `8회 남음`
- 예약 1건 직접 취소: `reservation_cancelled` 본문에 `9회 남음`
- 다시 예약: `reservation_created` 본문에 `8회 남음`
- 대기 예약이 확정됨: `waitlist_promoted` 본문에 현재 잔여 횟수 안내
- 취소 문의 승인으로 복원됨: `cancel_request_approved` 본문에 복원 후 잔여 횟수 안내

## 대상 범위 요약

- 스튜디오 전체 알림: 해당 스튜디오 활성 회원 전체
- 수업 영향 알림: 해당 수업에 실제로 연결된 회원
- 수강권/회원 상태 알림: 해당 회원 1명
