# 수강권 기반 스튜디오 출결/예약 앱 기획서
## 성인 취미 스튜디오용 (요가 / 필라테스 / 발레 등)

---

## 1. 서비스 개요

본 서비스는 요가, 필라테스, 발레 등 성인 취미 스튜디오를 위한 수강권 기반 예약 및 출결 관리 시스템이다.

서비스는 크게 두 영역으로 구성된다.

- 관리자 웹
  - 스튜디오 운영자가 사용하는 웹 백오피스
  - 수업 템플릿 관리, 수강권 상품 관리, 실제 수업 일정 생성, 회원 등록, 수강권 발급, 취소/환불/휴강/공지 관리 등을 담당
- 사용자 앱
  - 회원이 사용하는 모바일 앱
  - 회원가입, 본인 회원 ID 확인, 스튜디오에서 발급받은 수강권 확인, 달력에서 수업 예약, 예약 취소, 예약/수강권 이력 조회, 공지/이벤트 확인 등을 담당

이 서비스의 핵심은 단순 출석 체크가 아니라 다음 네 가지를 정확히 관리하는 것이다.

- 수업 일정 운영
- 회원 예약
- 수강권 차감 예정 / 차감 완료 관리
- 스튜디오 측의 운영 예외 처리

---

## 2. 핵심 운영 철학

### 2-1. 스튜디오 중심 운영
회원이 임의로 스튜디오와 연결되는 구조가 아니라, 스튜디오가 회원을 등록하는 구조로 간다.

즉:
- 사용자는 앱에 가입만 가능
- 스튜디오 연결은 스튜디오 관리자가 직접 수행
- 회원은 자기 회원 ID를 스튜디오에 알려주고, 스튜디오가 관리자 웹에서 등록

### 2-2. 예약 기반 차감 예정, 수업 종료 후 자동 차감
회원이 수업을 예약하면 즉시 차감 예정 상태가 된다.
수업이 종료되면, 취소되지 않은 예약은 자동으로 차감 완료 처리된다.
차감 예정의 총 합은 수강권의 잔여 횟수를 넘을 수 없다. 
즉, 수강권 횟수를 넘게 예약할 수 없다.

즉:
- 예약 = 차감 예정
- 취소 승인 = 예정 복원
- 수업 종료 = 자동 차감 완료

예약만 하고 오지 않아도 차감된다.

### 2-3. 사용자 앱은 단순하게, 관리자 웹은 운영 중심으로
사용자 앱은 예약과 조회 중심으로 단순하게 유지한다.
운영 예외 처리는 관리자 웹에서 담당한다.

예:
- 24시간 이내 취소 승인/거절
- 환불 처리
- 휴강 처리
- 회원 등록
- 수강권 발급

## 3. 관리자 계정 구조

관리자는 일반 회원처럼 자유롭게 가입하는 구조가 아니다.
관리자 계정은 시스템 운영자가 사전 발급하는 구조로 간다.

### 3-1. 관리자 계정 생성 방식
스튜디오에서 운영 시작을 원하면, 운영 측에 아래 정보를 전달한다.

- 스튜디오명
- 관리자 로그인용 이메일 또는 아이디
- 연락처
- 주소
- 기타 기본 스튜디오 정보

그 후 시스템에서는 다음을 처리한다.

1. 스튜디오 레코드 생성
2. 해당 스튜디오에 연결된 관리자 계정 생성
3. 초기 임시 비밀번호 생성
4. 관리자에게 계정 정보 전달

전달 정보 예시:
- 관리자 로그인 ID
- 임시 비밀번호
- 스튜디오명

### 3-2. 최초 로그인 Flow
1. 관리자가 관리자 웹 로그인
2. 임시 비밀번호로 최초 로그인
3. 비밀번호 변경 화면으로 강제 이동
4. 새 비밀번호 설정
5. 이후 정상 사용 가능

### 3-3. 관리자 계정 정책
- 관리자 계정은 공개 회원가입 없음
- 관리자가 로그인하면 본인 계정에 연결된 스튜디오 ID가 자동 적용
- 관리자 화면에서는 스튜디오 ID는 수정 불가
- 해당 관리자는 자기 스튜디오 데이터만 조회/수정 가능

---

## 4. 사용자 계정 구조

사용자는 앱에서 자유롭게 회원가입 가능하다.

### 4-1. 회원가입 시 생성되는 정보
- 사용자 기본 계정
- 사용자 프로필
- 회원 ID

### 4-2. 회원 ID 정책
회원 ID는 사용자가 앱에 가입하는 순간 자동 발급된다.

규칙:
- 길이: 5자리
- 문자 구성: 소문자 + 숫자
- 랜덤 생성
- 중복 불가
- 수정 불가

예:
- a3k9m
- 9q2x1
- b7n4c

### 4-3. 중복 체크 정책
회원가입 시 회원 ID를 랜덤 생성할 때,
이미 존재하는 회원 ID인지 확인한 뒤 중복이면 다시 생성한다.

즉:
1. 1차 랜덤 생성
2. 중복 검사
3. 중복이면 재생성
4. 유일할 때 저장

### 4-4. 사용자 입장에서의 흐름
1. 앱 회원가입
2. 로그인 완료
3. 앱 내 프로필에서 본인의 회원 ID 확인
4. 스튜디오에 해당 ID를 알려줌
5. 스튜디오가 관리자 웹에서 해당 회원 ID를 검색/등록

---

## 5. 스튜디오와 회원 연결 방식

이제 초대코드는 사용하지 않는다.
스튜디오와 회원 연결은 회원 ID 기반 수동 등록으로 한다.

### 5-1. 연결 주체
연결의 주체는 회원이 아니라 스튜디오가다.

즉:
- 회원이 앱 가입
- 본인의 회원 ID 확인
- 스튜디오에 회원 ID 전달
- 스튜디오 관리자가 관리자 웹에서 등록

### 5-2. 연결 구조
한 스튜디오은 여러 회원을 가질 수 있고,
한 회원은 여러 스튜디오에 등록될 수 있다.

따라서 실제 구조는:
- studio
- user
- studio_user_membership

형태의 연결 구조로 관리하는 것이 적합하다.

### 5-3. 여러 스튜디오 등록 가능
한 회원은 여러 스튜디오에 등록될 수 있다.

예:
- A 요가 스튜디오
- B 필라테스 스튜디오
- C 성인 발레 스튜디오

사용자는 앱에서 현재 확인하고 싶은 스튜디오를 선택할 수 있어야 한다.

스튜디오 선택 후 바뀌는 정보:
- 보유 수강권
- 열린 수업 달력
- 내 예약
- 공지사항
- 이벤트

## 6. 관리자 웹 초기 운영 순서

관리자의 실제 초기 세팅 순서는 아래와 같다.

1. 관리자 로그인
2. 수업 템플릿 등록
3. 수강권 상품 등록
4. 수업 템플릿을 기간 단위로 실제 달력에 일괄 반영
5. 회원 등록
6. 회원에게 수강권 발급
7. 이후 예약/취소/문의/환불/휴강/공지 운영

---

## 7. 수업 템플릿 등록

수업 템플릿은 실제 달력에 수업을 생성하기 위한 기본 규칙이다.

### 7-1. 수업 템플릿 포함 정보
- 스튜디오 ID (자동 연결, 수정 불가)
- 수업명
- 카테고리 (요가 / 필라테스 / 발레 등)
- 설명
- 요일 정보
- 시작 시간
- 종료 시간
- 기본 정원

예:
- 화/목 19:00 필라테스 베이직
- 월/수/금 20:00 요가 입문
- 토 11:00 성인 발레 초급

### 7-2. 등록 Flow
1. 관리자 웹 > 수업 템플릿 관리
2. 새 수업 템플릿 등록
3. 요일, 시간, 정원, 설명 입력
4. 저장

### 7-3. 예약 가능 여부
예약 가능 여부는 별도 필드로 두지 않는다.

실제 예약 가능 여부는 아래 기준으로 판단한다.
- 해당 날짜에 회차가 생성되어 있는가
- 정원이 남아 있는가
- 같은 시간대 중복 예약이 아닌가

---

## 8. 수강권 상품 등록

수강권 상품은 스튜디오에서 판매하는 상품 정의이다.

### 8-1. 수강권 상품 포함 정보
- 스튜디오 ID (자동 연결, 수정 불가)
- 수강권 이름
- 기본 횟수
- 유효기간 (x일 로 기입. 이는 사용자에게 부여될 때 부여한 날 기준으로 +x 일을 하여 실제 유효기간을 선정하기 위함)
- 판매 금액
- 설명
- 이 수강권으로 예약 가능한 수업 템플릿 목록

### 8-2. 등록 Flow
1. 관리자 웹 > 수강권 상품 관리
2. 새 수강권 등록
3. 이름, 횟수, 유효기간, 금액, 설명 입력
4. “예약 가능한 수업 선택” 영역에서 현재 스튜디오의 수업 템플릿 버튼 목록 표시
5. 해당 수업 템플릿들을 다중 선택
6. 저장

### 8-3. 예시
- 요가 8회권 → 요가 입문, 요가 중급
- 필라테스 10회권 → 필라테스 베이직, 필라테스 코어
- 통합 12회권 → 요가 입문, 필라테스 베이직, 성인 발레 초급

## 9. 실제 수업을 달력에 일괄 생성

관리자는 수업 템플릿을 실제 달력에 일정으로 반영해야 한다.

### 9-1. 방식
- 수업 템플릿 선택
- 기간 선택
- 해당 기간 동안 템플릿의 요일 규칙에 맞는 날짜들에 자동 생성

예:
수업 템플릿:
- 화/목 19:00 필라테스 베이직

적용 기간:
- 2026-03-01 ~ 2026-03-31

결과:
- 3월의 모든 화요일, 목요일에 해당 수업 회차 생성

### 9-2. Flow
1. 관리자 웹 > 일정 관리 / 달력 관리
2. “템플릿 일정 적용” 클릭
3. 수업 템플릿 선택
4. 적용 시작일 / 종료일 선택
5. 시스템이 요일 규칙에 따라 자동 생성
6. 생성 결과 확인
7. 저장

## 10. 개별 회차 수정 / 삭제

일괄 생성 후에도, 특정 날짜만 수정/삭제할 수 있어야 한다.

### 10-1. 필요한 이유
- 공휴일
- 강사 일정 변경
- 특정 날짜만 시간 변경
- 특정 날짜만 휴강

### 10-2. UX
달력 또는 일정 리스트에서 각 회차마다:
- 수정 버튼
- X 버튼

### 10-3. 개별 수정 예시
- 3월 12일 화요일 수업만 19:00 → 20:00
- 특정 날짜만 정원 변경

### 10-4. 개별 삭제 예시
- 공휴일이라 특정 날짜 수업만 제거

## 11. 회원 등록

회원은 앱에 가입한 뒤 자기 회원 ID를 스튜디오에 알려준다.
스튜디오은 관리자 웹에서 해당 회원을 등록한다.

### 11-1. Flow
1. 회원이 앱 가입 후 본인 회원 ID 확인
2. 스튜디오 데스크 또는 관리자에게 회원 ID 전달
3. 관리자 웹 > 회원 등록/회원 연결 메뉴 진입
4. 회원 ID 검색
5. 검색 결과 회원 확인
6. 현재 스튜디오에 등록
7. 등록 완료

### 11-2. 결과
- 해당 회원은 이 스튜디오의 회원으로 연결됨
- 관리자 웹 회원 목록에 노출됨
- 이후 수강권 발급 가능

---

## 12. 회원에게 수강권 발급

스튜디오에 등록된 회원에게만 수강권 발급 가능하다.

### 12-1. Flow
1. 관리자 웹 > 회원 관리
2. 회원 선택
3. 회원 상세 화면 진입
4. 수강권 발급 버튼 클릭
5. 이 스튜디오의 수강권 상품 중 하나 선택
6. 시작일 / 만료일 / 금액 확인 
- 시작일은 선택 가능하며, 만료일은 선택한 수강권의 유효기간 일수만큼이 더해져서 자동으로 산정되어 나타난다 (물론 관리자가 직접 수정 가능)
7. 저장

### 12-2. 결과
- 사용자 앱의 수강권 화면에 표시
- 해당 수강권으로 예약 가능한 수업들이 활성화됨

## 13. 취소 요청 처리

정책:
- 수업 시작 24시간 전까지는 회원이 직접 취소 가능
- 24시간 이내는 회원이 직접 취소 불가
- 관리자 문의를 통해 처리

### 13-1. 관리자 Flow
1. 관리자 웹 > 취소 요청 관리
2. 요청 목록 확인
   - 회원명
   - 회원 ID
   - 수업명
   - 날짜/시간
   - 수강권명
   - 요청 사유
3. 승인 또는 거절

### 13-2. 승인 시
- 예약 취소
- 차감 예정 복원
- 요청자에게 취소 완료 알림
- 정원 자리 복원
- 대기자 자동 승급되지 않음 (24시간 이내의 승급 건에 대해서는 자동 승급이 아닌 관리자 수동 승급 처리되어야 함. 대기자에게 직접 연락 후 대기자가 컨펌하면 이후 승인하여 대기자의 수강권 차감이 이루어질 수 있도록 해야한다)
- 승급자 승인 이후 승인자에게 푸시 알림

### 13-3. 거절 시
- 예약 유지
- 수업 종료 후 자동 차감
- 요청자에게 거절 안내

## 14. 환불 처리

환불은 회원이 직접 처리하지 않고 관리자가 직접 처리한다.

### 14-1. Flow
1. 관리자 웹 > 회원 관리
2. 회원 상세 진입
3. 보유 수강권 목록 확인
4. 환불할 수강권 선택
5. 환불 처리 버튼 클릭
6. 환불 금액, 사유 입력
7. 저장

### 14-2. 결과
- 수강권 상태는 refunded 또는 inactive 처리
- 해당 수강권은 더 이상 예약 불가
- 환불 금액은 통계에 반영
- 과거 이력은 유지

## 16. 사용자 앱 Flow

### 16-1. 앱 회원가입
1. 앱 설치
2. 회원가입
3. 로그인
4. 프로필 생성
5. 5자리 회원 ID 자동 발급
6. 프로필에서 회원 ID 확인 가능

안내 문구 예시:
- “스튜디오에 회원 ID를 전달하면 스튜디오 등록이 완료됩니다.”
- “스튜디오 등록이 완료되면 수강권이 표시됩니다.”

### 16-2. 여러 스튜디오 선택
회원이 여러 스튜디오에 등록된 경우, 현재 확인할 스튜디오를 선택할 수 있어야 한다.

선택 후 바뀌는 정보:
- 달력 수업 목록
- 내 예약
- 보유 수강권
- 공지사항
- 이벤트

### 16-3. 수강권 목록 화면
- 종료일 임박 순 정렬
- 이미 종료된 수강권은 맨 아래
- 종료된 수강권은 회색 처리

각 카드에 표시할 정보:
- 수강권 이름
- 총 횟수
- 잔여 횟수
- 차감 예정 횟수
- 차감 완료 횟수
- 시작일
- 종료일

### 16-4. 수강권 상세 화면
- 수강권 이름
- 총 횟수
- 잔여 / 예정 / 완료
- 시작일 / 종료일
- 예약 가능한 수업 종류 라벨
- 시간순 사용 이력

이력 항목 예시:
- 2026-03-12 19:00 필라테스 베이직 — 예정
- 2026-03-10 20:00 요가 입문 — 완료
- 2026-03-08 19:00 요가 입문 — 취소 복원

### 16-5. 달력 화면
- 이번 달 / 다음 달 최대 2개월 조회
- 날짜별 수업 리스트 간단 노출
- 시간 / 수업명 / 잔여 자리 수 / 예약 여부 표시

### 16-6. 수업 상세 및 예약
- 아직 예약 안 한 경우: 등록 버튼
- 이미 예약한 경우
  - 24시간 이전: 취소 버튼
  - 24시간 이내: 관리자 문의 문구 또는 비활성

예약 시:
- 사용 가능한 내 수강권 목록 노출
- 기본값은 만료일이 가장 빠른 수강권
- 사용자가 변경 가능

### 16-7. 대기 등록 및 자동 승급
- 정원 가득 찬 경우 대기 등록
- 취소로 자리가 나면 먼저 신청한 순으로 자동 승급
- 승급 시 푸시 알림

### 16-8. 내 예약 화면
탭 예시:
- 예정
- 대기
- 완료/지난 수업
- 취소 요청/취소됨

### 16-9. 최종 정책 요약
- 정원 남으면 즉시 예약
- 정원 차면 대기 등록
- 수업 시작 전까지만 예약 가능
- 같은 시간대 중복 예약 불가
- 24시간 전까지 무료 취소 가능
- 24시간 이내 직접 취소 불가
- 예약 시 차감 예정
- 수업 종료 후 취소되지 않은 예약은 자동 차감 완료
- 예약 후 미참석도 차감
